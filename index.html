Roblox EditableMesh Module

Softbody Physics for Roblox

Create squishy, deformable sphere and mesh softbodies with runtime physics, collision, custom settings, culling, and mesh conversion utilities.

Get Started API Reference

Install

Place the `SoftbodyHandler` module inside `ReplicatedStorage`, then require it from a client or server script depending on your use case. 

local ReplicatedStorage = game:GetService("ReplicatedStorage")

local softbody_handler = require(ReplicatedStorage.SoftbodyHandler)

Quick Start

Create a basic sphere softbody.

local ReplicatedStorage = game:GetService("ReplicatedStorage")

local softbody_handler = require(ReplicatedStorage.SoftbodyHandler)

softbody_handler.Create(Vector3.new(0, 20, 0), Color3.fromRGB(120, 185, 255), {
	Radius = 5,
	Subdivisions = 2,
	ShapeStiffness = 0.32,
	BendStiffness = 0.05,
	FloorY = 0,
})

API Reference

softbody_handler.Create

Creates a new softbody and returns the body object.

softbody_handler.Create(spawnPosition, color, settings)
Parameter Type Description
`spawnPosition` `Vector3` World position where the softbody spawns.
`color` `Color3` Color used by generated sphere softbodies.
`settings` `table?` Optional configuration table. 
local body = softbody_handler.Create(Vector3.new(0, 20, 0), Color3.new(1, 0, 0), {
	Radius = 6,
	Subdivisions = 2,
	Bounce = 0.15,
})

softbody_handler.Destroy

Destroys a softbody and removes it from the simulation list.

softbody_handler.Destroy(body)
local body = softbody_handler.Create(Vector3.new(0, 20, 0), Color3.new(1, 0, 0))

task.wait(10)
softbody_handler.Destroy(body)

softbody_handler.Step

Steps all active softbodies. This is usually connected automatically with `RunService.Heartbeat`. 

softbody_handler.Step(dt)

softbody_handler.GetBodies

Returns the active softbody list. Useful for debug tools or custom interactions.

local bodies = softbody_handler.GetBodies()

utility.from_mesh_part

Converts a Roblox `MeshPart` into mesh data that can be passed into `softbody_handler.Create`. 

local utility = require(ReplicatedStorage.SoftbodyHandler.Utility)

local meshData = utility.from_mesh_part(workspace.Cube)

softbody_handler.Create(Vector3.new(15, 20, 0), workspace.Cube.Color, {
	Mesh = meshData,
})

Settings

Setting Type Default Description
`Radius``number``5`Sphere radius for generated softbodies.
`Subdivisions``number``2`Mesh detail. Higher values cost much more.
`Damping``number``0.98`Velocity damping for Verlet integration.
`Gravity``Vector3``Vector3.new(0, -140, 0)`Gravity applied each physics step.
`ShapeStiffness``number``0.1`How strongly the body returns to its original shape.
`BendStiffness``number``0.1`Strength of bend constraints.
`CornerSoftness``number``0.6`Softens stretched bend constraints.
`Friction``number``1`Collision friction.
`Bounce``number``2`Collision bounce response.
`FloorY``number``0`Fallback floor height.
`FixedDt``number``1 / 60`Fixed physics timestep.
`MaxStepsPerFrame``number``2`Max physics catch-up steps per frame.
`RandomAngularVelocity``number``0`Adds random starting tumble.
`AngularVelocity``Vector3``Vector3.zero`Initial spin direction and speed.
`Mesh``table``nil`Custom mesh data from `utility.from_mesh_part`.
`ExtraIgnore``table``nil`Parts ignored by softbody collision.
`AlwaysRenderDistance``number``80`Always render when camera is within this distance.
`MaxRenderDistance``number``250`Hide/sleep beyond this distance.
`CullPadding``number``200`Extra screen padding before culling.

Custom Mesh Softbody

Use `Utility.from_mesh_part` to turn a low-poly `MeshPart` into softbody mesh data. High-poly meshes can lag or hit Roblox EditableMesh memory limits. 

local ReplicatedStorage = game:GetService("ReplicatedStorage")

local softbody_handler = require(ReplicatedStorage.SoftbodyHandler)
local utility = require(ReplicatedStorage.SoftbodyHandler.Utility)

local cube = workspace:WaitForChild("Cube")
local meshData = utility.from_mesh_part(cube)

local body = softbody_handler.Create(Vector3.new(15, 20, 0), cube.Color, {
	Mesh = meshData,
	ShapeStiffness = 0.45,
	BendStiffness = 0.04,
	FloorY = 0,
})

cube.Transparency = 1
cube.CanCollide = false

Examples

Random Softbody Sphere

softbody_handler.Create(Vector3.new(0, 30, 0), Color3.fromRGB(120, 185, 255), {
	Radius = 5,
	Subdivisions = 2,
	RandomAngularVelocity = 4,
	FloorY = 0,
})

Multiple Softbodies

for i = 1, 8 do
	local x = ((i - 1) % 4) * 14
	local z = math.floor((i - 1) / 4) * 14

	softbody_handler.Create(Vector3.new(x, 30, z), Color3.fromRGB(120, 185, 255), {
		Radius = 4,
		Subdivisions = 1,
		FloorY = 0,
	})

	task.wait(0.1)
end

Performance Notes

Softbodies are expensive because every vertex is simulated and updated. Keep meshes low-poly, limit active softbodies, and avoid high subdivision counts.

  • Use `Subdivisions = 1` for many spheres.
  • Use `Subdivisions = 2` for better quality but fewer active objects.
  • Avoid high-poly imported meshes.
  • Use culling settings for large scenes.
  • Destroy unused softbodies with `softbody_handler.Destroy(body)`.
Roblox EditableMesh has memory limits. If creation fails, reduce mesh complexity or the amount of active softbodies.
